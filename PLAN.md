# 检测报告 PDF 自动识别入库方案

## Summary

- v1 不做网页，工作入口放在飞书多维表格；管理员在“检测报告主表”上传 PDF。
- 后端用 Python 脚本部署在云服务器，监听/轮询多维表格新增或待解析记录，下载 PDF 后解析并回填。
- AI 只用于开发解析规则、生成脚本和辅助调试；最终生产识别链路不用大模型，采用 PDF 文本解析 + 本地 OCR + 规则/模板提取。
- 数据落在多维表格中，采用“检测报告主表 + 检测项目明细表 + 模板规则表”的结构。

## Key Changes

- 建 3 张核心多维表格：
  - `检测报告主表`：PDF附件、报告编号、样品名称、检测机构、委托单位/客户、报告日期、报告结论、产品/项目分类、解析状态、错误原因、解析模板、上传人、上传时间、关联明细。
  - `检测项目明细表`：关联报告、检测项目、检测值、单位、标准要求/限值、检测方法、单项结论、来源页码、来源文本片段。
  - `模板规则表`：检测机构、模板名称、识别关键词、规则版本、启用状态、备注。
- 解析流程：
  - 管理员上传 PDF 后，记录状态置为 `待解析`。
  - Python 服务获取附件并下载到临时目录。
  - 先用 PDF 解析库提取文本；如果文本为空、乱码严重或关键字段缺失，再走本地 OCR。
  - 根据机构名、报告标题、关键词匹配模板规则。
  - 按模板规则提取主表字段和检测项目明细。
  - 写回主表，并批量新增明细表记录。
  - 失败时不自动入库脏数据，只把状态改为 `解析失败`，写入错误原因，并通过飞书机器人通知管理员。
- 多模板处理：
  - 第一阶段收集现有真实 PDF 样本，按检测机构/版式归类。
  - 每类模板单独维护解析规则，规则版本记录到主表，方便追溯。
  - 新模板无法匹配时，状态置为 `待新增模板`，机器人通知管理员补充样本。
- 权限设计：
  - 管理员：可上传 PDF、查看全部字段、修改解析状态、整理文件和模板规则。
  - 客服：通过客服视图查看允许展示的报告基础字段和结论，不直接维护模板规则。
  - 研发：通过研发视图查看报告基础字段和检测项目明细，可看更多技术字段。
  - 不只依赖“筛选视图”当安全边界；需要配置飞书表格/字段权限。若飞书原生权限无法满足严格隔离，则用脚本同步出“客服可见表”和“研发可见表”。

## Implementation Plan

- 准备飞书侧：
  - 创建检测报告多维表格应用。
  - 建主表、明细表、模板规则表。
  - 配置管理员、客服、研发三类角色的表权限、字段权限和视图。
  - 创建飞书机器人，用于发送解析成功、失败、待新增模板等通知。
- 准备后端服务：
  - 云服务器部署 Python 服务。
  - 配置飞书开放平台应用权限：读取多维表格、读取附件、写入记录、批量新增记录、发送机器人通知。
  - 保存配置项：飞书 app token、table id、field id、机器人 webhook、临时文件目录。
- 解析模块：
  - `pdf_text_parser`：处理可复制文本 PDF。
  - `ocr_parser`：处理扫描件，本地 OCR 优先用 PaddleOCR 或 Tesseract。
  - `template_detector`：根据关键词和版式特征识别模板。
  - `field_extractor`：提取报告编号、样品名称、检测机构、日期、结论等主表字段。
  - `item_extractor`：提取检测项目、检测值、单位、标准要求、结论等明细字段。
  - `feishu_writer`：写回主表，批量写入明细表，并维护关联字段。
- 运行方式：
  - v1 可用定时轮询，每 1-5 分钟扫描 `待解析` 记录，稳定后再升级为事件触发。
  - 每次解析必须记录日志：record_id、PDF 文件名、模板、状态、错误原因、耗时。
  - 同一报告编号重复上传时，默认不覆盖旧记录，标记为 `疑似重复` 并通知管理员。

## Test Plan

- 用现有真实 PDF 样本建立测试集，至少覆盖：
  - 可复制文本 PDF。
  - 扫描件 PDF。
  - 多页报告。
  - 多检测机构模板。
  - 表格型检测项目。
  - 解析失败或未知模板。
  - 重复报告编号。
- 验收标准：
  - 主表字段能正确回填。
  - 每条检测项目能正确写入明细表并关联主表。
  - 解析失败不会产生错误明细数据。
  - 管理员、客服、研发看到的字段范围符合权限要求。
  - 飞书机器人能正确通知成功、失败、未知模板。
- 首版建议先用 20-50 份历史报告做离线测试，再接入真实上传流程。

## Assumptions

- 第一版按现有样本定制字段和模板规则；没有样本时，先使用通用主表字段作为最低版本。
- 扫描件使用本地 OCR，不把报告内容发给大模型。
- 自动解析失败时只通知管理员，不做完整人工复核工作流。
- 最终数据库采用飞书多维表格；如果后续数据量、查询性能或权限隔离要求变高，再增加独立数据库作为底层存储。
